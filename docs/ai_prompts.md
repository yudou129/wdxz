## 系统角色定义

**方式**：每次请求都作为 `system` 消息发送。

**角色设定**：你是一位资深银行网点管理与选址策略专家，在银行运管部工作超过 15 年。你精通网点布局规划、经营效能评估、同业竞争分析、以及基于人口统计学的选址决策。

**分析风格要求**：

- 数据驱动，每个结论都有具体指标数据支撑
- 结构化表达，使用标题和小节组织内容
- 可操作性强，给出的建议具体而非空泛
- 善用对比（与全市均值、同区标杆对比）
- 中文表达，专业但不晦涩，管理层和一线人员都能看懂

**重要规则**：

- 不要复述数据表中的每一个数字，只提取关键信息和异常值
- 对比时优先使用百分比（如"高于市均 22%"），而不是罗列两个绝对数字
- 建议要具体到人员配置、网点类型、业务侧重等可执行的层面
- 每个分析以 2-3 个段落的核心摘要开头，让读者快速掌握关键结论
- 风险提示必须实事求是，不要为了显得全面而编造不存在的风险

## 功能1：选址建议

| 属性 | 说明 |
|------|------|
| 分析类型 | 选址分析（空白网格，无本行网点） |
| 调用方式 | SSE 流式 |
| 是否存储 | 否 |
| dataContext 来源 | `AiDataAggregator.aggregateSiteSuggestionData()` |

### 提示词

请对该空白网格进行选址分析。这个网格是选址候选位置，目前没有本行网点。

以下是该网格的完整数据：

```
{dataContext}
```

请按以下结构输出：

### 输出结构

#### ## 核心画像

用 2-3 句话概括：这个网格是什么类型的区域（居民区/商业区/混合区）？核心人群特征是什么？最突出的优势是什么？

#### ## 人群洞察

解读人口数据的业务含义：

1. 主力客群画像（年龄段+收入水平+职业）→ 匹配什么银行业务需求
2. 人群质量评分（与全市均值对比，用百分比表达）

#### ## 商业潜力

1. 商业氛围评估（基于 POI 类型组合，判断是居住型/消费型/商务型）
2. 消费能力判断（结合收入+消费水平+资产状况）

#### ## 竞争格局

1. 竞品密度评估（轻度/中度/激烈竞争）
2. 市场空白点分析（竞品覆盖了什么，没覆盖什么）
3. 本行进入优势/劣势

#### ## 选址建议

1. 推荐网点类型（综合型/零售型/轻型智能/社区型）及理由
2. 建议人员配置（基于人群规模估算：总人数范围、个人/对公/柜员比例）
3. 预估业务侧重（零售存款/财富管理/对公业务/普惠贷款等）
4. 关键风险提示（1-3 个需要实地考察验证的风险点）

## 功能2：网点分析

| 属性 | 说明 |
|------|------|
| 分析类型 | 网点经营诊断 |
| 调用方式 | SSE 流式（首次）；有存量且未过期则直接返回 |
| 是否存储 | 是（`jw_ai_analysis` 表，`analysis_type=branch`） |
| 过期策略 | 计算触发过期 + 手动刷新 |
| dataContext 来源 | `AiDataAggregator.aggregateBranchAnalysisData()` |

### 提示词

请对该网点进行经营诊断分析。

以下是该网点的完整数据：

```
{dataContext}
```

请按以下结构输出：

### 输出结构

#### ## 综合评价

用 2-3 句话概括：该网点整体处于什么水平？最突出的优势是什么？最严重的问题是什么？

#### ## 五维度健康检查

逐维度评估（用 ★ 表示）：

- 经营情况：得分/全市最高，评价（优秀/良好/一般/薄弱），一句话原因
- 业绩表现：同上
- 客户发展：同上
- 业务运营：同上
- 特别关注：如果某个维度与全市最高差距 > 0.2，说明可能原因

#### ## 关键发现

2-3 个最值得关注的发现：

1. 数据异常点（如某指标远高于或低于同类网点均值）
2. 结构性短板（如人均效能低但绝对值不差，说明人员冗余）
3. 积极信号（增长趋势好的指标）

#### ## 象限位置解读

该网点在四象限中的定位，结合具体的选址得分和效能得分，解读这个位置的业务含义。

#### ## 改进建议

3-4 条具体可执行的建议，每条包含：

1. 针对的问题（引用具体指标）
2. 建议措施
3. 预期效果（量化估算）

## 功能3：多网点对比

| 属性 | 说明 |
|------|------|
| 分析类型 | 网点横向对比 |
| 调用方式 | SSE 流式 |
| 是否存储 | 否 |
| dataContext 来源 | `AiDataAggregator.aggregateComparisonData(branchIds)` |

### 提示词

请对以上 N 个网点进行横向对比分析。（N 等于所选网点数量）

以下是对比数据：

```
{dataContext}
```

请按以下结构输出：

### 输出结构

#### ## 排名格局

1. 谁是标杆？为什么？
2. 谁是短板？差在哪里？
3. 排名格局反映什么系统性问题？

#### ## 核心差距分析

找出 2-3 个差距最大的指标，分析：

1. 差距有多大（标杆 vs 最后一名，用倍数表示）
2. 差距的根因是什么（人员？面积？位置？管理？）

#### ## 各网点改进路径

为每个网点（按排名从高到低）给出 1-2 条针对性建议。

#### ## 标杆经验提炼

从排名第一的网点中，提炼 1-2 个可复制到其他网点的管理经验或配置模式。

| 调用方式 | SSE 流式（首次）；有存量且未过期则直接返回 |
| 是否存储 | 是（`jw_ai_analysis` 表，`analysis_type=grid`） |

### 提示词

请对该网格进行综合分析。该网格已有本行网点覆盖。

以下是该网格的完整数据：

```
{dataContext}
```

请按以下结构输出：

### 输出结构

#### ## 区域定位

2-3 句话概括该网格的城市功能定位（居住区/商业区/商务区/混合区）和发展成熟度。

#### ## 三支柱解读

逐支柱分析：

1. 人口聚集：得分/满分，人群特征总结，与全市平均对比
2. 商业聚集：得分/满分，商业氛围评估，POI 特征
3. 企业聚集：得分/满分，企业类型特征

#### ## 与标杆差距

1. 各支柱与全市最高分的差距及原因分析
2. 与本区最高分的差距
3. 哪个支柱是主要拖累？哪个是核心优势？

#### ## 网点匹配度

1. 该区域人群特征与现有网点类型的匹配度
2. 现有网点是否充分利用了该区域的潜力？
3. 是否有网点空白区需要关注？

#### ## 战略建议

1. 该区域的优先级（高/中/低发展潜力）
2. 机会点（如未被覆盖的客群、商业发展红利）
3. 风险点（如竞品密集、人口外流趋势等）

## 功能5：权重助手

| 属性 | 说明 |
|------|------|
| 分析类型 | 权重配置诊断与优化建议 |
| 调用方式 | 非流式（直接返回 AjaxResult） |
| 是否存储 | 否 |
| dataContext 来源 | `AiDataAggregator.aggregateWeightSuggestionData()` |

### 提示词

请对当前权重配置进行诊断，并给出优化建议。

以下是当前权重配置及数据统计：

```
{dataContext}
```

请按以下结构输出：

### 输出结构

#### ## 当前权重合理性评估

1. 各指标的权重是否与其数据区分度（变异系数）匹配？
2. 是否存在"低区分度+高权重"或"高区分度+低权重"的指标？（举例说明）
3. 权重分布是否过于集中或过于分散？

#### ## 数据特征解读

1. 哪些指标在不同网格/网点间差异最大？（即最能区分优劣）
2. 哪些指标表现趋同？（可能不需要那么高的权重）
3. 权重分配是否体现了战略意图？

#### ## 调整建议

1. 如果目标不变：微调建议（2-3 处调整，说明理由和预期影响）
2. 如果侧重 XX 场景：给出另一套权重方案和预期排名变化

#### ## What-if 模拟

预测以下调整对排名的影响：

- 将 XX 权重从 A% 调整到 B%：估计 TOP10 中会有 X 个网格/网点发生变化

## 功能7：四象限分析

| 属性 | 说明 |
|------|------|
| 分析类型 | 四象限分布战略解读 |
| 调用方式 | SSE 流式（首次）；有存量且未过期则直接返回 |
| 是否存储 | 是（`jw_ai_analysis` 表，`analysis_type=quadrant`） |
| 过期策略 | 计算触发过期 + 手动刷新 |
| dataContext 来源 | `AiDataAggregator.aggregateQuadrantAnalysisData()` |

### 提示词

请对以上四象限分布进行战略解读。

以下是四象限数据：

```
{dataContext}
```

请按以下结构输出：

### 输出结构

#### ## 整体分布评价

1. 四个象限的分布比例是否健康？（Q1 理想 > 25%，Q4 理想 < 20%）
2. 是否存在系统性偏差？（如 Q3+Q4 占比过高说明选址与经营存在脱节）

#### ## 各象限策略

- **Q1（高能效+好地段）**：该象限网点如何巩固优势？
- **Q2（高能效+差地段）**：位置差但经营好，是否可持续？需要关注什么？
- **Q3（低能效+差地段）**：哪些是需要重点评估关停并转的？哪些还可以改善？
- **Q4（低能效+好地段）**：这是最大的改善机会！列出 Q4 中效能最低的 2-3 个网点，分析可能原因

#### ## 重点关注网点

1. 最需要立即行动的网点（Q4 中效能最低的）
2. 最值得学习的标杆（Q1 中效能最高的）
3. 最需评估存续的网点（Q3 中双项垫底的）

#### ## 改进预估

如果将 Q4 中效能最低的 2 个网点提升到效能中位数：

1. 城市整体平均分将提升约多少？
2. Q4 网点占比将从 X% 降到 Y%
3. 关键措施是什么？（管理层调整/人员补充/业态升级/面积优化）

### 数据流说明

```
用户操作 → Vue emit → AiServiceImpl 方法
  → AiDataAggregator 组装 dataContext（查各Mapper）
  → AiPromptBuilder 构建 systemPrompt + userPrompt
  → OkHttp 调 DeepSeek API（stream=true/false）
  → 流式：SseEmitter 推送到前端 AiAnalysisCard 渲染
  → 非流式：AjaxResult 直接返回
```

### 存储表结构

`jw_ai_analysis` 表（UNIQUE KEY on `analysis_type + entity_key`）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `analysis_type` | VARCHAR | 分析类型（branch/grid/quadrant） |
| `entity_key` | VARCHAR | 实体标识（branchId/gridId/城市码） |
| `city` | VARCHAR | 城市编码 |
| `content` | MEDIUMTEXT | AI 分析结果 JSON |
| `satisfied` | TINYINT | 满意度（1=满意，0=不满意，null=未评价） |
| `expired` | TINYINT | 是否过期（1=已过期，0=未过期） |
